## Architecture Principles

- Functional core, imperative shell.
- Core rule evaluation is side-effect free.
- Value objects are preferred over primitive strings and numbers at domain boundaries.
- Important rule logic should prefer function composition and typed strategies over broad conditional branches.
- Rule identifiers use UUID v7.
- Canonical rule storage is expected to be JSON AST, with human-friendly file DSLs compiled into that format later.
- Mocking frameworks are avoided by default; use fakes, stubs, and real domain objects.
- Infrastructure tests should use real boundaries such as Testcontainers where practical.
- Tidy First is preferred for behavior-preserving structural improvements.
- Ktlint owns formatting; detekt should avoid duplicate formatting checks.
- Architecture checks can be added with ArchUnit once service, SDK, and adapter modules exist.
